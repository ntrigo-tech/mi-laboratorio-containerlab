Manual de Laboratorio Definitivo: CCNA Enterprise Core

Este documento interactivo representa la guía completa de configuración, verificación y solución de fallas bajo el sistema operativo Cisco IOS para el simulador Cisco Packet Tracer. Está estructurado de forma incremental, abarcando las competencias mandatorias del examen de certificación de Cisco.

📌 Tips clave para el examen CCNA: Recordá que Cisco IOS no es sensible a las mayúsculas en los comandos, pero sí en los argumentos (como nombres de VLAN o contraseñas). Podés usar abreviaciones como conf t en lugar de configure terminal y la tecla Tab para autocompletar.

Fase 1: Inicialización, Seguridad de Dispositivos y Manejo de Usuarios

El aseguramiento del plano de control es el primer paso en cualquier despliegue de red. Aquí se configuran los modos de ejecución, usuarios locales con hashes criptográficos y el banner de advertencia legal.

Cisco IOS Configuración Base y Acceso Protegido

Objetivo: Configurar el Router desde el modo de usuario hasta el modo global, aplicando contraseñas seguras y creando cuentas administrativas con privilegios completos.

Router> enable
Router# configure terminal

# Asignar nombre de host identificatorio
Router(config)# hostname R1

# Proteger el acceso al modo privilegiado (Privileged EXEC) con algoritmo hash tipo 9
R1(config)# enable secret CiscoCCNA123

# Crear usuario administrador con máximo nivel de privilegios y contraseña encriptada
R1(config)# username admin privilege 15 secret AdminPass456

# Forzar la encriptación de todas las contraseñas planas en el archivo de texto plano
R1(config)# service password-encryption

# Configurar el Banner de Advertencia Legal obligatorio
R1(config)# banner motd # ACCESO RESTRINGIDO. SOLO PERSONAL AUTORIZADO #

# Configurar el tiempo de espera de sesión (Timeout) y evitar interrupciones de logs en consola
R1(config)# line console 0
R1(config-line)# exec-timeout 5 0
R1(config-line)# logging synchronous
R1(config-line)# login local
R1(config-line)# exit
🔍 Comandos de Verificación de Estado Base

Ejecutá estos comandos desde el modo privilegiado (R1#):

# Ver toda la configuración activa en la memoria RAM
R1# show running-config

# Ver la configuración persistente guardada en la NVRAM
R1# show startup-config

# Guardar los cambios de la RAM a la NVRAM (Crucial para el examen)
R1# copy running-config startup-config   (o simplemente: write memory)

Fase 2: Conmutación de Capa 2 (VLANs Estáticas y Troncalización)

La segmentación lógica permite mitigar tormentas de broadcast y aplicar seguridad perimetral a nivel de accesos físicos.

Switching Creación de VLANs y Asignación de Puertos

Objetivo: Acceder al **Switch S1** de Packet Tracer, dar de alta la base de datos de VLANs y asignar interfaces físicas a sus respectivos dominios de colisión.

Switch> enable
Switch# configure terminal
Switch(config)# hostname S1

# Crear las tres VLANs del laboratorio
S1(config)# vlan 10
S1(config-vlan)# name Usuarios
S1(config-vlan)# vlan 20
S1(config-vlan)# name Gestion
S1(config-vlan)# vlan 30
S1(config-vlan)# name Servidores
S1(config-vlan)# exit

# Asignar puertos de acceso de forma estática
S1(config)# interface fastEthernet 0/1
S1(config-interface)# switchport mode access
S1(config-interface)# switchport access vlan 10

S1(config)# interface fastEthernet 0/2
S1(config-interface)# switchport mode access
S1(config-interface)# switchport access vlan 20

# Asignación múltiple (Rango) para la zona de servidores
S1(config)# interface range fastEthernet 0/3 - 4
S1(config-if-range)# switchport mode access
S1(config-if-range)# switchport access vlan 30
S1(config-if-range)# exit

# Configurar el puerto Troncal (Trunk) hacia el Router R1 bajo estándar 802.1Q
S1(config)# interface gigabitEthernet 0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk allowed vlan 10,20,30
S1(config-if)# exit
🔍 Verificación de Tablas de Capa 2 (Switching) 
# Verificar la asignación de puertos y nombres de la base de datos de VLANs
S1# show vlan brief

# Comprobar qué interfaces están operando como troncales activos
S1# show interfaces trunk

# Inspeccionar la tabla de Direcciones MAC (CAM Table) del Switch
S1# show mac address-table

Fase 3: Enrutamiento Inter-VLAN (Router-on-a-Stick) y Servidor DHCP

El Router actúa como la pasarela de comunicación (Gateway) entre las VLANs, procesando el etiquetado 802.1Q y proveyendo direccionamiento dinámico de forma centralizada.

Routing Configuración de Subinterfaces y Ámbitos DHCP

Objetivo: Configurar la interfaz troncal de R1, levantar pasarelas secundarias de Capa 3 y habilitar los pools de DHCP excluyendo las direcciones estáticas asignadas a administración y servidores.

R1# configure terminal

# Excluir direcciones de la asignación dinámica para evitar duplicidades e IP Conflicts
R1(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.10
R1(config)# ip dhcp excluded-address 192.168.20.1 192.168.20.20

# Crear el Pool DHCP para la VLAN 10 (Usuarios)
R1(config)# ip dhcp pool POOL_VLAN10
R1(dhcp-config)# network 192.168.10.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.10.1
R1(dhcp-config)# dns-server 8.8.8.8
R1(dhcp-config)# exit

# --- CONFIGURACIÓN DE ROUTER-ON-A-STICK ---
# Encender la interfaz física principal (¡No lleva dirección IP!)
R1(config)# interface gigabitEthernet 0/0/0
R1(config-if)# no shutdown
R1(config-if)# exit

# Subinterfaz Lógica VLAN 10
R1(config)# interface gigabitEthernet 0/0/0.10
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0

# Subinterfaz Lógica VLAN 20
R1(config)# interface gigabitEthernet 0/0/0.20
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0

# Subinterfaz Lógica VLAN 30
R1(config)# interface gigabitEthernet 0/0/0.30
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 192.168.30.1 255.255.255.0
R1(config-subif)# exit
🔍 Comandos de Validación de Direccionamiento y Arrendamientos 
# Ver el estado lógico y físico de las subinterfaces creadas
R1# show ip interface brief

# Monitorear qué direcciones IP ha entregado dinámicamente el Router
R1# show ip dhcp binding

# Ver estadísticas del servidor DHCP e identificar caídas de mensajes (Drops)
R1# show ip dhcp server statistics

Fase 4: Enrutamiento Dinámico con OSPFv2 Single Area

Para interconectar este laboratorio con el resto de la corporación imaginaria, habilitamos OSPFv2 de Área Única (Área 0) para el intercambio automático de prefijos de red.

Routing Dinámico Activación de OSPFv2 y Pasivación de Interfaces

Objetivo: Configurar el proceso OSPFv2, declarar los segmentos del laboratorio usando máscaras comodín (Wildcard Masks) y aplicar seguridad de pasivación para evitar fugas de paquetes de enrutamiento hacia las PCs.

R1(config)# router ospf 1
# Definir el Router ID para desambiguar la topología (Formato IP)
R1(config-router)# router-id 1.1.1.1

# Declarar los segmentos que van a participar del proceso OSPF en el Área 0
R1(config-router)# network 192.168.10.0 0.0.0.255 area 0
R1(config-router)# network 192.168.20.0 0.0.0.255 area 0
R1(config-router)# network 192.168.30.0 0.0.0.255 area 0

# --- SEGURIDAD CCNA: PASSIVE INTERFACES ---
# Evita el envío de paquetes OSPF de tipo HELLO hacia las redes de usuarios
R1(config-router)# passive-interface gigabitEthernet 0/0/0.10
R1(config-router)# passive-interface gigabitEthernet 0/0/0.20
R1(config-router)# passive-interface gigabitEthernet 0/0/0.30
R1(config-router)# exit
🔍 Diagnóstico de Protocolos de Enrutamiento (OSPF)

Comandos indispensables de troubleshooting en el examen CCNA:

# Ver la tabla de enrutamiento IP completa (Las rutas OSPF se marcan con la letra 'O')
R1# show ip route

# Verificar los vecinos OSPF descubiertos y su estado de sincronización (Debe decir FULL/BDR o FULL/DR)
R1# show ip ospf neighbor

# Ver los parámetros generales de OSPF (ID, Timers, Interfaces Pasivas)
R1# show ip protocols
