Laboratorio 2: Enrutamiento Inter-VLAN y Direccionamiento Híbrido

Este laboratorio práctico profundiza en la segmentación de redes corporativas. Se implementarán tres dominios de difusión independientes utilizando un switch de acceso para cada VLAN, concentrando el enrutamiento en un Router central (Router-on-a-Stick). El esquema de direccionamiento combina hosts estáticos con asignación dinámica por DHCP gestionada por el propio router.

📌 Concepto de Diseño: Aunque se utilicen switches físicos separados para cada departamento, la configuración de troncales (Trunks) y el etiquetado de tramas bajo el estándar IEEE 802.1Q siguen siendo obligatorios para que el Router pueda identificar y enrutar el tráfico de cada subred de forma aislada.

Fase 1: Diseño de la Topología y Plan de Direccionamiento IP

Antes de interactuar con la CLI, es fundamental estructurar las conexiones físicas y el mapa de direccionamiento del escenario.

Topología Componentes y Matriz de Red

Componentes requeridos en el lienzo:

  • 1 Router (R1): Modelo ISR 4331.
  • 3 Switches (S1, S2, S3): Modelo clásico 2960 de 24 puertos.
  • 3 PCs Finales (PC-V10, PC-V20, PC-V30): Una conectada a cada switch.

Tabla de Direccionamiento Lógico

VLAN Nombre Red / Máscara Gateway (Router) Dispositivo (Puerto) Tipo de IP
VLAN 10 Ingenieria 192.168.10.0 /24 192.168.10.1 PC-V10 (En S1 Fa0/1) Estática (192.168.10.10)
VLAN 20 Administracion 192.168.20.0 /24 192.168.20.1 PC-V20 (En S2 Fa0/1) Estática (192.168.20.10)
VLAN 30 Invitados 192.168.30.0 /24 192.168.30.1 PC-V30 (En S3 Fa0/1) Dinámica (DHCP Pool)

Esquema de Interconexión de Cables (Cables Directos)

Router R1 (Gi0/0/0) ──────> Puerto Gi0/1 en Switch S1 (Troncal)
Router R1 (Gi0/0/1) ──────> Puerto Gi0/1 en Switch S2 (Troncal)
Router R1 (Gi0/0/2) ──────> Puerto Gi0/1 en Switch S3 (Troncal)

Fase 2: Scripts de Configuración de Dispositivos

Accedé a la CLI de cada equipo y aplicá las siguientes plantillas de comandos para establecer el plano de datos.

CLI Configuración en los Switches de Acceso

Switch 1 (S1 - Red de Ingeniería)

Switch> enable
Switch# configure terminal
Switch(config)# hostname S1
S1(config)# vlan 10
S1(config-vlan)# name Ingenieria
S1(config-vlan)# exit

# Puerto de acceso para la PC
S1(config)# interface fastEthernet 0/1
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 10

# Enlace Troncal hacia el Router
S1(config)# interface gigabitEthernet 0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk allowed vlan 10
S1(config-if)# exit

CLI Configuración de Router (Gateway & DHCP Server)

En R1 se habilitan las interfaces físicas y se levanta el servicio de asignación automática de parámetros IP exclusivamente para la VLAN 30.

Router> enable
Router# configure terminal
Router(config)# hostname R1

# --- CONFIGURACIÓN DE SERVIDOR DHCP PARA VLAN 30 ---
# Excluir las primeras direcciones para uso de administración/infraestructura
R1(config)# ip dhcp excluded-address 192.168.30.1 192.168.30.10

# Crear el pool de asignación
R1(config)# ip dhcp pool POOL_INVITADOS
R1(dhcp-config)# network 192.168.30.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.30.1
R1(dhcp-config)# dns-server 8.8.8.8
R1(dhcp-config)# exit

# --- CONFIGURACIÓN DE INTERFACES EN R1 ---
# Interfaz hacia S1 (VLAN 10)
R1(config)# interface gigabitEthernet 0/0/0
R1(config-if)# no shutdown
R1(config-if)# interface gigabitEthernet 0/0/0.10
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0
R1(config-subif)# exit

# Interfaz hacia S2 (VLAN 20)
R1(config)# interface gigabitEthernet 0/0/1
R1(config-if)# no shutdown
R1(config-if)# interface gigabitEthernet 0/0/1.20
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0
R1(config-subif)# exit

# Interfaz hacia S3 (VLAN 30)
R1(config)# interface gigabitEthernet 0/0/2
R1(config-if)# no shutdown
R1(config-if)# interface gigabitEthernet 0/0/2.30
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 192.168.30.1 255.255.255.0
R1(config-subif)# exit

Fase 3: Pruebas de Diagnóstico y Verificación Obligatorias

Para garantizar el éxito del despliegue en un entorno real o de examen, se deben ejecutar de manera rigurosa las siguientes pruebas en los dispositivos finales y de red.

Troubleshooting Control de Calidad del Laboratorio

Prueba 1: Verificación del Arrendamiento DHCP (PC-V30)

Ingresá al Escritorio de la PC-V30 > IP Configuration y conmutá el botón a DHCP. Debería obtener de inmediato la IP 192.168.30.11, la máscara 255.255.255.0 y el Gateway 192.168.30.1.

💬 Comando de Consola en Windows/PC CLI

Abrí el Command Prompt de la PC-V30 y ejecutá:

C:\> ipconfig
# Si no toma IP correctamente, forzá la renovación con:
C:\> ipconfig /renew

Prueba 2: Diagnóstico de la Tabla de Concesiones en el Router

Para comprobar desde la perspectiva del plano de control que el router entregó los datos correctamente:

R1# show ip dhcp binding
# Debe listar la dirección MAC de la PC-V30 asociada a la IP 192.168.30.11

Prueba 3: Trazabilidad y Conectividad Extremo a Extremo (Ping & Traceroute)

Abrí la terminal de comandos de la PC-V10 (IP Estática 192.168.10.10) e intentá alcanzar a la PC de la otra VLAN para verificar que R1 está conmutando los paquetes:

C:\> ping 192.168.20.10   # Ping hacia PC-V20
C:\> ping 192.168.30.11   # Ping hacia PC-V30 (DHCP)

Para visualizar de forma exacta el salto que da el paquete a través de la subinterfaz del Router:

C:\> tracert 192.168.30.11

El primer salto de la lista debe ser de manera mandatoria la IP del Gateway local (192.168.10.1).
