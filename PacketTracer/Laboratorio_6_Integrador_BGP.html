Laboratorio 6 (Proyecto Integrador): Interconexión de Sistemas Autónomos con eBGP

Este laboratorio final consolida la totalidad de los conceptos vistos en la cursada: Conmutación de Capa 2 (VLANs, Trunks), Servicios Core (DHCP, DNS), Enrutamiento Interior (OSPFv2 para Router-on-a-Stick) y Enrutamiento de Borde Exterior (eBGP). Simularemos dos organizaciones independientes en Internet que deciden intercambiar sus prefijos privados de forma dinámica.

🌐 Arquitectura BGP (Border Gateway Protocol): A diferencia de OSPF que busca la ruta más rápida basándose en el costo del enlace (métrica de estado de enlace), BGP es un protocolo de vector de ruta (Path Vector) que toma decisiones basándose en políticas, atributos y saltos de Sistemas Autónomos (AS). Es el protocolo utilizado para interconectar ISPs y grandes redes corporativas en el mundo real.

Fase 1: Mapa General de la Topología y Direccionamiento

A continuación se detalla la distribución de los bloques de red. El escenario se divide en dos Sistemas Autónomos (AS) unidos por un enlace troncal punto a punto.

Topología Distribución de Hardware en Packet Tracer

Bloque A: Sistema Autónomo 100 (Sede Central Corp)

  • Router R1 (AS 100): Concentra las VLANs corporativas locales mediante Router-on-a-Stick en su puerto Gi0/0/0 e interconecta la WAN en la Gi0/0/3. El enrutamiento interno de sus segmentos se resuelve vía OSPF.
  • Switch S1 (Central): Configurado con Trunk hacia R1 y puertos de acceso para VLAN 10 (Ingeniería), VLAN 20 (Administración) y VLAN 30 (Servidor DHCP corporativo).

Bloque B: Sistema Autónomo 200 (Nueva Red Sucursal / Socio)

  • Router R2 (AS 200): Actúa como el Gateway perimetral de la nueva sucursal. Configura Router-on-a-Stick en su interfaz Gi0/0/0 para sus propios segmentos y conecta la WAN en la Gi0/0/3.
  • Switch S2 (Sucursal): Maneja dos nuevas VLANs de acceso independiente:
    • VLAN 40 (Cómputo): Red 192.168.40.0/24 (PC-V40 conectada a puerto de acceso).
    • VLAN 50 (Servidores Locales): Red 192.168.50.0/24 (PC-V50 / Servidor local conectado a puerto de acceso).

Fase 2: Configuración del Nuevo Sistema Autónomo (AS 200)

Para garantizar que la sucursal funcione internamente de forma óptima, aplicaremos la técnica de Router-on-a-Stick en el Switch S2 y el Router R2.

CLI Paso A: Base de datos de VLANs y Trunk en Switch S2
Switch> enable
Switch# configure terminal
Switch(config)# hostname S2

# Crear VLANs de la sucursal
S2(config)# vlan 40
S2(config-vlan)# name Computo_Sucursal
S2(config-vlan)# vlan 50
S2(config-vlan)# name Servidores_Sucursal
S2(config-vlan)# exit

# Asignar accesos de hosts
S2(config)# interface fastEthernet 0/10
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 40

S2(config)# interface fastEthernet 0/20
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 50
S2(config-if)# exit

# Configurar Troncal hacia el nuevo Router R2
S2(config)# interface gigabitEthernet 0/1
S2(config-if)# switchport mode trunk
S2(config-if)# do write memory
CLI Paso B: Configuración de Subinterfaces en Router R2
Router> enable
Router# configure terminal
Router(config)# hostname R2

# Levantar interfaz física base
R2(config)# interface gigabitEthernet 0/0/0
R2(config-if)# no shutdown
R2(config-if)# exit

# Subinterfaz VLAN 40
R2(config)# interface gigabitEthernet 0/0/0.40
R2(config-subif)# encapsulation dot1Q 40
R2(config-subif)# ip address 192.168.40.1 255.255.255.0
R2(config-subif)# exit

# Subinterfaz VLAN 50
R2(config)# interface gigabitEthernet 0/0/0.50
R2(config-subif)# encapsulation dot1Q 50
R2(config-subif)# ip address 192.168.50.1 255.255.255.0
R2(config-subif)# exit

# Configurar interfaz WAN física hacia AS 100
R2(config)# interface gigabitEthernet 0/0/3
R2(config-if)# ip address 10.0.0.2 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# exit
R2(config)# do write memory

Fase 3: Configuración del Enrutamiento eBGP Core

Con las interfaces operativas y comunicación en el enlace WAN (comprobable mediante un ping directo entre R1 y R2), procederemos a configurar la sesión eBGP para el intercambio dinámico de prefijos IP.

CLI Configuración de BGP en el Router Corporativo R1 (AS 100)

Declararemos el número de sistema autónomo local, estableceremos la vecindad apuntando a la IP WAN de R2 (indicando su AS remoto) y anunciaremos los bloques de red internos corporativos.

R1> enable
R1# configure terminal

# Levantar el proceso BGP para el Sistema Autónomo 100
R1(config)# router bgp 100
R1(config-router)# bgp log-neighbor-changes

# Declarar al vecino (IP de la interfaz de R2 y su AS correspondiente)
R1(config-router)# neighbor 10.0.0.2 remote-as 200

# Anunciar las redes internas a Internet de forma exacta
R1(config-router)# network 192.168.10.0 mask 255.255.255.0
R1(config-router)# network 192.168.20.0 mask 255.255.255.0
R1(config-router)# network 192.168.30.0 mask 255.255.255.0
R1(config-router)# exit
R1(config)# do write memory
CLI Configuración de BGP en el Router Sucursal R2 (AS 200)

Realizaremos el proceso espejo en la sucursal. Al confirmar el comando del vecino, el protocolo negociará la adyacencia de forma inmediata.

R2> enable
R2# configure terminal

# Levantar el proceso BGP para el Sistema Autónomo 200
R2(config)# router bgp 200
R2(config-router)# bgp log-neighbor-changes

# Declarar al vecino (IP de la interfaz de R1 y su AS correspondiente)
R2(config-router)# neighbor 10.0.0.1 remote-as 100

# Anunciar las redes internas de la sucursal
R2(config-router)# network 192.168.40.0 mask 255.255.255.0
R2(config-router)# network 192.168.50.0 mask 255.255.255.0
R2(config-router)# exit
R2(config)# do write memory

Al finalizar la configuración, las consolas CLI de ambos routers dispararán un mensaje informativo del sistema (Syslog) indicando que la adyacencia pasó a estado operativo: %BGP-5-ADJCHANGE: neighbor 10.0.0.x Up.

Fase 4: Diagnóstico, Verificación y Convergencia Global

BGP tiene comandos de diagnóstico específicos. Validaremos el plano de control antes de realizar las pruebas lógicas de extremo a extremo.

Troubleshooting Comandos de Auditoría Perimetral

1. Verificar el Resumen de Vecinos BGP

Para comprobar el estado de la sesión, la cantidad de prefijos recibidos y el tiempo de adyacencia activa, ejecutá en cualquier router:

R1# show ip bgp summary

Asegurate de que la columna de la derecha (State/PfxRcd) muestre un número entero (la cantidad de redes aprendidas) y **no** palabras como Active o Idle, lo cual indicaría una falla de negociación.

2. Inspeccionar la Tabla de Enrutamiento (Prefijo "B")

Ejecutemos el comando global en R1 para validar que los caminos dinámicos inter-AS ya se inyectaron en el plano de reenvío:

R1# show ip route

Verás las redes de la sucursal precedidas de forma explícita por la letra **B**:

B    192.168.40.0/24 [20/0] via 10.0.0.2, 00:05:14
B    192.168.50.0/24 [20/0] via 10.0.0.2, 00:05:14

Nota de examen CCNA: La Distancia Administrativa (DA) para las rutas aprendidas por conexiones **eBGP es de 20**, lo que las hace altamente prioritarias frente a protocolos internos como OSPF (110).

3. Prueba del Plano de Datos Extremo a Extremo

Abrí la consola de comandos de la PC-V10 (VLAN 10 en AS 100) y tirale un ping a la PC-V40 (VLAN 40 en AS 200):

C:\> ping 192.168.40.10

La traza de paquetes responderá con total éxito, demostrando que dos redes gobernadas de forma independiente y aisladas físicamente por diferentes Sistemas Autónomos pueden comunicarse transparentemente gracias a la inyección dinámica de BGP.
