Guía de Iniciación: Cisco Packet Tracer y Comandos Base Telecomunicaciones B

Este módulo interactivo cubre los pasos iniciales obligatorios para adentrarse en el ecosistema de redes Cisco. Aprenderás a gestionar tu entorno de simulación, construir topologías físicas básicas de Capa 2 y Capa 3, e interactuar con la consola de comandos de Cisco IOS.

📌 Tip de Navegación de Consola: Dentro de Cisco IOS, podés usar el signo de pregunta ? en cualquier momento para ver los comandos disponibles en tu modo actual o para completar los argumentos de un comando parcial (ej: show ip ?).

Paso 1: Alta de Cuenta y Acceso a la Plataforma

Cisco centraliza el acceso a sus simuladores mediante sus plataformas de aprendizaje oficiales. Es necesario contar con un perfil activo para evitar el modo restringido de guardado.

Plataforma Registro y Desbloqueo del Software
  1. Ingresá al sitio web oficial de Skills for All (skillsforall.com) o Cisco Networking Academy (netacad.com).
  2. Inscribite de forma gratuita en el curso introductorio "Getting Started with Cisco Packet Tracer". Esto habilitará la sección de descargas oficiales en tu perfil.
  3. Descargá e instalá la versión correspondiente a tu sistema operativo (Windows, Linux o macOS).
  4. Al iniciar el programa por primera vez, seleccioná la opción de inicio de sesión con tus credenciales de Skills for All o NetAcad para desbloquear el entorno de forma permanente.

Paso 2: Construcción de la Topología Física

El primer laboratorio consta de una estructura clásica de red de área local (LAN) conectada a un dispositivo de distribución y pasarela.

Lienzo Dispositivos y Esquema de Conexiones

Arrastrá y posicioná los siguientes elementos desde la barra de herramientas inferior de Packet Tracer hacia el espacio de trabajo:

  • Router (R1): Modelo ISR 4331 (Ubicado en Network Devices > Routers).
  • Switch (S1): Modelo clásico 2960 de 24 puertos (Ubicado en Network Devices > Switches).
  • Dispositivos Finales: Dos PCs genéricas (PC 1 y PC 2) y un Servidor (Server 1) (Ubicados en End Devices).

Esquema de Cableado (Cable Directo de Cobre / Copper Straight-Through)

PC 1 (FastEthernet 0)        ───────────────>  Switch S1 (FastEthernet 0/1)
PC 2 (FastEthernet 0)        ───────────────>  Switch S1 (FastEthernet 0/2)
Server 1 (FastEthernet 0)    ───────────────>  Switch S1 (FastEthernet 0/3)
Router R1 (GigabitEthernet 0/0/0) ──────────>  Switch S1 (GigabitEthernet 0/1)

Configuración IP Estática en Servidor 1

Hacé clic sobre Server 1, ingresá a la solapa Desktop > IP Configuration, tildá Static y completá los parámetros de red:

  • IP Address: 192.168.30.10
  • Subnet Mask: 255.255.255.0
  • Default Gateway: 192.168.30.1

Nota: Las interfaces de PC 1 y PC 2 deben quedar configuradas en modo DHCP de manera predeterminada.

Paso 3: Modos de Ejecución y Comandos de Usuario en Cisco IOS

Hacé clic sobre el Router R1 e ingresá a la solapa CLI. Si se despliega el asistente inicial de configuración, escribí no y presioná Enter para acceder a la terminal limpia.

Cisco IOS Flujo Secuencial de Consola

1. Modo Usuario (User EXEC Mode)

Modo de lectura restringido. Representado por el prompt con el signo mayor que (>).

Router>
Router> ?   # Despliega la lista reducida de comandos permitidos en este nivel

2. Modo Privilegiado (Privileged EXEC Mode)

Permite ver estadísticas de rendimiento, configuraciones guardadas y realizar diagnósticos detallados. Representado por el signo numeral (#).

Router> enable
Router#

3. Modo de Configuración Global

Requerido para modificar parámetros operativos que impacten en el comportamiento general del chasis. Entorno delimitado por (config)#.

Router# configure terminal
Router(config)# 
Router(config)# hostname R1       # Cambia el nombre identificatorio del dispositivo

4. Modo de Configuración de Interfaz

Submodo destinado a aplicar directivas específicas sobre un puerto físico o lógico seleccionado.

R1(config)# interface gigabitEthernet 0/0/0
R1(config-if)# no shutdown       # Enciende eléctricamente la interfaz física seleccionada
R1(config-if)# exit
R1(config)# exit
R1#

Paso 4: Verificación y Persistencia de la Configuración

Todo cambio realizado en la CLI impacta de manera inmediata en la memoria volátil del equipo, pero se perderá ante un corte de energía si no se consolida en el almacenamiento persistente.

Almacenamiento RAM vs. NVRAM y Auditoría

Comandos Esenciales de Verificación (Modo Privilegiado #)

Antes de guardar, es mandatorio auditar que los cambios aplicados en caliente sean los correctos:

# 1. Ver la configuración activa operando en la memoria RAM
R1# show running-config

# 2. Ver el estado simplificado y direccionamiento de todas las interfaces
R1# show ip interface brief

# 3. Ver la configuración de respaldo almacenada en la memoria NVRAM (No volátil)
R1# show startup-config

Cómo Guardar la Configuración de Forma Permanente

Para transferir el contenido de la memoria volátil activa (RAM) hacia la persistente de arranque (NVRAM), disponés de dos comandos alternativos:

# Método Estándar y Oficial (Recomendado para exámenes de certificación)
R1# copy running-config startup-config
Destination filename [startup-config]?   # Presioná ENTER para confirmar

# Método Corto Tradicional (Ampliamente usado en la industria)
R1# write memory   # (o simplemente: wr)
💡 Tips Críticos de Operación sobre Guardado y Verificación:
  • El peligro de un reinicio accidental: Si modificás configuraciones críticas (como cambiar contraseñas o apagar un puerto) y el equipo se apaga o se ejecuta el comando reload antes de guardar, el dispositivo arrancará con el estado anterior guardado en la NVRAM.
  • El atajo de ejecución (Do): Si estás metido muy adentro en los modos de configuración (ej: R1(config-if)#) y querés tirar un comando de verificación o guardar sin salir a la raíz, anteponé la palabra clave do (ej: do show ip interface brief o do wr).
  • Validar la NVRAM vacía: Si un equipo es nuevo o se le borró la configuración con el comando erase startup-config, al ejecutar show startup-config la consola devolverá el mensaje: "startup-config is not present". Es completamente normal hasta que guardes por primera vez.
