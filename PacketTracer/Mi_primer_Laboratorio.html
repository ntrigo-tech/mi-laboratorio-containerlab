De OSI a los comandos que usás en el CLI

Modelos de referencia, cableado físico, subnetting IPv4/IPv6 con ejercicios y simulador VLSM, conceptos clave, comandos IOS organizados por bloque temático, y cómo empezar a practicar en Packet Tracer.

Modelo OSI 7 capas

Modelo de referencia teórico que divide la comunicación de red en 7 capas independientes. Cada capa se comunica solo con la capa inmediatamente superior e inferior, y con su capa equivalente en el destino (comunicación entre pares).

7
AplicaciónApplication
Interfaz entre la red y las apps del usuario. Provee servicios de red directamente usables.HTTP, HTTPS, FTP, SMTP, DNS, Telnet, SSH
6
PresentaciónPresentation
Traduce, cifra y comprime los datos para que la capa de aplicación los entienda (formato común).SSL/TLS, JPEG, ASCII, EBCDIC, MPEG
5
SesiónSession
Establece, administra y termina sesiones de comunicación entre dos hosts (diálogo).NetBIOS, RPC, PPTP, sockets
4

Entrega extremo a extremo, control de flujo y errores. Segmenta los datos y usa puertos.TCP (confiable), UDP (best-effort)

🔗 Cable Directo (Straight-through)

Mismo estándar en ambos extremos (568B–568B). Conecta dispositivos de distinta capa/tipo.

PC ↔ Switch
Switch ↔ Router
Router ↔ Hub

🔀 Cable Cruzado (Crossover)

Un extremo 568A y el otro 568B (cruza Tx/Rx). Conecta dispositivos del mismo tipo.

PC ↔ PC
Switch ↔ Switch
Router ↔ Router
PC ↔ Router (directo)

Auto-MDIX: los switches y routers modernos detectan automáticamente el tipo de cable y ajustan los pines internamente, por lo que hoy en día casi nunca importa usar directo o cruzado. Aun así, sigue siendo pregunta obligada en cualquier certificación básica.

Subnetting IPv4 paso a paso

Subnetear es "tomar prestados" bits de la porción de host para crear varias subredes más pequeñas dentro de una red mayor.

Método rápido (fórmulas)

Subredes creadas

2n donde n = bits prestados al host.

Hosts útiles por subred

2h − 2 donde h = bits de host restantes (se restan red y broadcast).

Ejemplo resuelto: 192.168.1.0/24 → 4 subredes

Necesito 4 subredes → 2² = 4, tomo prestados 2 bits → nueva máscara /26 (255.255.255.192), incremento = 64.

Truco del "incremento" (método de resta 256): 256 − (valor de máscara en el octeto interesante) = salto entre redes. Ej: máscara .192 → 256−192=64 → las redes van de 64 en 64 (0, 64, 128, 192…).

VLSM — Variable Length Subnet Mask

En lugar de dar a todas las subredes el mismo tamaño, VLSM asigna a cada una solo los hosts que necesita, evitando desperdicio de direcciones. Regla de oro: siempre asignar primero el requerimiento más grande, para que los bloques encajen sin solaparse. Es la base de todo diseño IP profesional.

Resolvé mentalmente o en papel y después despliega la respuesta para verificar.

Ver solución

IPv6

Direcciones de 128 bits (vs. 32 bits de IPv4), escritas en 8 grupos hexadecimales de 16 bits separados por dos puntos. Diseñado para resolver el agotamiento de IPv4 y simplificar el enrutamiento.

Anatomía de una dirección

2001:0db8:acad:0001:0000:0000:0000:0001
Prefijo de sitio /48 (asignado por el ISP) ID de subred (16 bits, hasta /64) ID de interfaz (64 bits, host)

Reglas de compresión

1. Omitir ceros a la izquierda

0db8 → db8
0000 → 0

2. Doble dos puntos (::) — solo una vez

2001:0db8:acad:0001:0000:0000:0000:0001
→ 2001:db8:acad:1::1

Tipos de direcciones IPv6

TipoRango / prefijoUso
Unicast global2000::/3Equivalente a las IP públicas de IPv4, enrutable en Internet.
Link-localFE80::/10Automática en cada interfaz, solo válida en el segmento local (usada por NDP/vecinos, no se enruta).
Unique Local (ULA)FC00::/7Equivalente a las IP privadas RFC 1918 de IPv4, uso interno.
MulticastFF00::/8Un emisor, varios receptores. IPv6 no tiene broadcast, lo reemplaza por multicast.
Loopback::1/128Equivalente a 127.0.0.1.
Sin especificar::/128Ausencia de dirección (usada antes de obtener una).

Subnetting en IPv6

No hay escasez de direcciones, así que el enfoque cambia: normalmente el ISP asigna un /48 o /56 por sitio, y cada subred/VLAN usa un /64 (el estándar para que SLAAC funcione correctamente, ya que reserva 64 bits para el ID de interfaz). Subnetear IPv6 significa jugar con los bits entre el prefijo de sitio y el /64, no reducir el host como en IPv4.

Ejemplo: con un sitio 2001:db8:acad::/48, se pueden crear 216 = 65 536 subredes /64 distintas simplemente variando los 16 bits de "ID de subred" (2001:db8:acad:0001::/64, 2001:db8:acad:0002::/64, etc.) — sin necesidad de VLSM complejo.

DHCPv6Versión con estado, similar a DHCP en IPv4; puede ser "stateful" (asigna todo) o "stateless" (solo entrega DNS y otros datos, la IP la genera SLAAC).

Conceptos clave

Los fundamentos que sostienen todo lo demás: segmentación lógica, resolución de direcciones, diagnóstico y enrutamiento.

VLAN — Virtual LAN

Segmentación lógica de una LAN

Divide un dominio de broadcast físico en varios dominios de broadcast lógicos, independientemente de la ubicación física de los puertos. Mejora seguridad, rendimiento y organización (por departamento, ej. VLAN 10=Ventas, VLAN 20=IT).

Trunk (802.1Q): enlace entre switches que transporta múltiples VLANs etiquetando cada trama con el ID de VLAN. Access port: pertenece a una sola VLAN, para hosts finales. VLAN nativa: la única que viaja sin etiqueta por un trunk 802.1Q.

Router-on-a-stick / SVI (Switch Virtual Interface) permiten enrutar entre VLANs (inter-VLAN routing).

ARP — Address Resolution Protocol

Resuelve IP → MAC

Cuando un host conoce la IP destino pero no la MAC, envía un broadcast ARP Request ("¿quién tiene esta IP?"); el dueño responde con ARP Reply (unicast) indicando su MAC. El resultado se guarda en la tabla ARP (caché) para evitar repetir el proceso.

show arp / arp -a  ·  Gratuitous ARP: un host anuncia su propia IP/MAC sin que se lo pidan (detecta duplicados de IP). En IPv6, ARP se reemplaza por NDP.

ICMP — Internet Control Message Protocol

Mensajes de control y diagnóstico de Capa 3

No transporta datos de usuario; informa errores y condiciones de red. Base de ping (Echo Request/Reply) y traceroute (Time Exceeded + Destination Unreachable). No usa puertos TCP/UDP.

Tipos comunes: 0=Echo Reply, 8=Echo Request, 3=Destination Unreachable, 11=Time Exceeded, 5=Redirect. ICMPv6 además transporta NDP.

Protocolos de enrutamiento

Permiten que los routers construyan y compartan dinámicamente sus tablas de enrutamiento, en vez de configurar rutas estáticas manualmente.

RIP
Distance Vector · IGP
Métrica: saltos (máx 15). Simple, obsoleto, lento en converger (v1 sin VLSM, v2 sí).
IGRP / EIGRP
Distance Vector avanzado (híbrido) · IGP · Cisco propietario
Métrica compuesta (ancho de banda+delay). EIGRP usa DUAL, converge rápido, sucesor factible.
OSPF
Link State · IGP · Estándar abierto
Métrica: costo (basado en ancho de banda). Usa áreas (Area 0 backbone), algoritmo SPF (Dijkstra).
BGP
Path Vector · EGP
Protocolo de Internet entre Sistemas Autónomos (AS). Basado en políticas y atributos de ruta, no en métrica pura.
ProtocoloTipoConvergenciaUso típico
RIPv2Distance VectorLentaRedes pequeñas/laboratorio, ya casi no se usa en producción
EIGRPHíbrido (Cisco)RápidaRedes empresariales con equipos Cisco

DNS · DHCP · NAT

DNS — Domain Name System

Traduce nombres de dominio a direcciones IP

Sistema jerárquico y distribuido (root → TLD → dominio autoritativo). Usa UDP/53 para consultas normales y TCP/53 para transferencias de zona o respuestas grandes.

Registros: A (IPv4), AAAA (IPv6), CNAME (alias), MX (correo), PTR (resolución inversa), NS (servidor de nombres).

DHCP — Dynamic Host Configuration Protocol

Asigna configuración IP automáticamente

Proceso DORA: Discover (broadcast del cliente) → Offer (oferta del servidor) → Request (el cliente pide esa oferta) → Acknowledge (confirmación). Entrega IP, máscara, gateway, DNS y tiempo de concesión (lease).

IP Helper-address / DHCP Relay: reenvía broadcasts DHCP a un servidor en otra subred.

NAT — Network Address Translation

Traduce direcciones IP privadas ↔ públicas

NAT estático: mapeo 1 a 1 fijo. NAT dinámico: pool de IPs públicas asignadas bajo demanda. PAT (NAT overload): muchas IPs privadas comparten una sola IP pública usando números de puerto distintos — el más común en redes domésticas/empresariales.

Inside local / Inside global / Outside local / Outside global — la terminología clásica para NAT.

Otros conceptos que conviene dominar

STP (Spanning Tree Protocol)

Evita bucles de Capa 2 bloqueando enlaces redundantes. Elige un Root Bridge y calcula puertos root/designados/bloqueados. Variantes: PVST+, RSTP (802.1w), MST.

EtherChannel

Agrupa varios enlaces físicos en uno lógico para más ancho de banda y redundancia. Protocolos de negociación: PAgP (Cisco) y LACP (estándar).

ACL (Access Control List)

Filtra tráfico por IP/puerto/protocolo. Estándar (solo IP origen) vs Extendida (origen, destino, puerto, protocolo). Se procesan en orden, primera coincidencia gana.

HSRP / VRRP / GLBP

Protocolos de gateway redundante (First Hop Redundancy). Varios routers comparten una IP virtual; si el activo falla, el standby toma el control.

Subnetting / VLSM

División de una red en subredes de distinto tamaño para aprovechar el espacio de direcciones. Base de todo el diseño IP.

QoS

Prioriza cierto tráfico (voz, video) sobre otro mediante clasificación, marcado (DSCP/CoS), colas y policing/shaping.

PPP / Frame Relay / MPLS

Tecnologías WAN clásicas. PPP para enlaces punto a punto con autenticación (PAP/CHAP); MPLS en redes de proveedor moderno.

VPN / IPsec / GRE

GRE crea túneles genéricos; IPsec cifra y autentica el tráfico (AH/ESP). Combinados dan VPN site-to-site seguras.

Packet Tracer: qué es y cómo se usa

El simulador de redes gratuito que te permite armar y probar todo lo de esta guía sin comprar hardware real.

¿Qué es? Un simulador visual de redes gratuito, desarrollado por Cisco, que permite arrastrar routers, switches, PCs, servidores y otros dispositivos a un lienzo, cablearlos, configurarlos con el mismo CLI que un equipo real (IOS simulado), y ver el tráfico circular en tiempo real o paso a paso (modo simulación con paquetes visibles capa por capa).

¿Para qué sirve? Es la herramienta de práctica estándar en cualquier curso de networking: no reemplaza un lab real 100%, pero permite equivocarse, repetir y experimentar sin romper nada ni gastar en equipos. Todos los conceptos de esta guía (VLAN, OSPF, NAT, ACLs, etc.) se pueden armar y probar ahí.

Flujo de trabajo básico

1

Elegir dispositivos

En la barra inferior izquierda seleccionás la categoría (Routers, Switches, End Devices, WAN Emulation...) y arrastrás el modelo al lienzo (ej. Router 2911, Switch 2960).

2

Cablear (Connections)

Elegís el tipo de cable (cobre directo/cruzado, consola, fibra, serial) y hacés clic en cada dispositivo para seleccionar el puerto de origen y destino. Un punto verde en ambos extremos confirma el enlace activo.

3

Configurar

Clic en el dispositivo → pestaña CLI para escribir comandos IOS reales (los mismos de esta guía), o pestaña Config para hacerlo con menús gráficos sin tipear.

4

Probar conectividad

Desde una PC: Desktop → Command Prompt → ping. O usá el ícono de sobre ("Add Simple PDU") para enviar un paquete de prueba y verlo viajar por la topología.

5

Modo Simulación

Cambiás de Realtime a Simulation (esquina inferior derecha) para congelar el tiempo y avanzar paquete por paquete, viendo exactamente qué pasa en cada capa OSI — ideal para entender ARP, STP o el proceso DORA de DHCP en cámara lenta.
