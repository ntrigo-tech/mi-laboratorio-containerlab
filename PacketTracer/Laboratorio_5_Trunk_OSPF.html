Laboratorio 5: Optimización con Router-on-a-Stick y Enrutamiento Dinámico OSPFv2

Este laboratorio representa el estándar de diseño profesional en redes medianas. Reduciremos el costo físico de la topología anterior unificando el tráfico local a través de un único enlace troncal 802.1Q (Router-on-a-Stick). Asimismo, reemplazaremos las rutas estáticas configurando el protocolo de estado de enlace OSPF para automatizar la convergencia y el intercambio de prefijos IP con la WAN pública.

🌐 OSPF y la Máscara de Wildcard: A diferencia de las rutas estáticas, OSPF utiliza máscaras de Wildcard (el inverso de la máscara de red) para declarar qué interfaces del router van a participar del proceso de enrutamiento dinámico en el Área 0.

Fase 1: Rediseño Físico de la Topología (Ahorro de Cables)

Modificaremos la topología del Laboratorio 4. Eliminaremos los tres cables independientes que iban de R1 a cada switch y utilizaremos un Switch Central de Distribución para concentrar el tráfico.

Topología Estructura Jerárquica e Interconexión Troncal

Cómo reestructurar el escenario en Packet Tracer:

  1. Switch Central (S1): Actuará como el nodo concentrador. Conectá las PCs de las VLANs 10, 20 y el Server-DHCP de la VLAN 30 a los puertos de acceso de este único Switch.
  2. El Enlace Troncal: Tirá un **único cable directo** desde el puerto GigabitEthernet 0/1 de S1 hacia el puerto GigabitEthernet 0/0/0 del Router R1.
  3. Enlace WAN e ISP: Mantené el cable cruzado o directo punto a punto entre R1 (Gi0/0/3) e ISP (Gi0/0/0) en la red 10.0.0.0/30.

Esquema Lógico del Enlace Físico Unificado

                                     ┌─────── Subinterface Gi0/0/0.10 (VLAN 10) -> GW: 192.168.10.1
                                     ├─────── Subinterface Gi0/0/0.20 (VLAN 20) -> GW: 192.168.20.1
[Switch S1] ══════ Enlace Troncal ══>│[Router R1]
 (Gi0/1)          (Tag 802.1Q)       └─────── Subinterface Gi0/0/0.30 (VLAN 30) -> GW: 192.168.30.1
                                                       │
                                                 [Puerto Gi0/0/3] (WAN)

Fase 2: Configuración del Switch y Creación del Trunk

Debemos indicarle al Switch que agrupe sus puertos de acceso y que configure la interfaz que va al router como un puerto Troncal capaz de transportar múltiples IDs de VLAN.

CLI Configuración en Switch S1
Switch> enable
Switch# configure terminal
Switch(config)# hostname S1

# Crear la base de datos de VLANs
S1(config)# vlan 10
S1(config-vlan)# name Ingenieria
S1(config-vlan)# vlan 20
S1(config-vlan)# name Administracion
S1(config-vlan)# vlan 30
S1(config-vlan)# name Invitados
S1(config-vlan)# exit

# Asignar los puertos de acceso locales para las PCs y Servidor
S1(config)# interface fastEthernet 0/10
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 10

S1(config)# interface fastEthernet 0/20
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 20

S1(config)# interface fastEthernet 0/30
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 30
S1(config-if)# exit

# --- CONFIGURAR EL PUERTO TRONCAL HACIA EL ROUTER R1 ---
S1(config)# interface gigabitEthernet 0/1
S1(config-if)# description Enlace_Troncal_Hacia_R1
S1(config-if)# switchport mode trunk
S1(config-if)# do write memory

Fase 3: Configuración del Router R1 (Router-on-a-Stick)

En el router no asignaremos IPs a la interfaz física Gi0/0/0. En su lugar, crearemos subinterfaces virtuales por software y les aplicaremos el encapsulamiento 802.1Q correspondiente a cada VLAN.

CLI Creación de Subinterfaces en R1
R1> enable
R1# configure terminal

# 1. Encender la interfaz física base (DEBE ESTAR SIN IP)
R1(config)# interface gigabitEthernet 0/0/0
R1(config-if)# no shutdown
R1(config-if)# exit

# 2. Crear subinterface para VLAN 10
R1(config)# interface gigabitEthernet 0/0/0.10
R1(config-subif)# description Gateway_VLAN_10
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0
R1(config-subif)# exit

# 3. Crear subinterface para VLAN 20
R1(config)# interface gigabitEthernet 0/0/0.20
R1(config-subif)# description Gateway_VLAN_20
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0
R1(config-subif)# exit

# 4. Crear subinterface para VLAN 30
R1(config)# interface gigabitEthernet 0/0/0.30
R1(config-subif)# description Gateway_VLAN_30
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 192.168.30.1 255.255.255.0
R1(config-subif)# exit

# 5. Configurar el puerto WAN físico (Igual que el Lab 4)
R1(config)# interface gigabitEthernet 0/0/3
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit

Fase 4: Activación del Enrutamiento Dinámico OSPFv2

¡Atención! Eliminá cualquier ruta estática previa si reciclaste el laboratorio anterior (con no ip route...). Ahora configuraremos OSPF con Process ID 1 para que los routers dialoguen entre sí en el área única de Backbone (Área 0).

CLI Configuración OSPF en Router R1

Anunciaremos las tres subredes de nuestras VLANs y el enlace WAN utilizando las máscaras Wildcard respectivas:

R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1

# Declarar las redes locales y la WAN dentro del Área 0
R1(config-router)# network 192.168.10.0 0.0.0.255 area 0
R1(config-router)# network 192.168.20.0 0.0.0.255 area 0
R1(config-router)# network 192.168.30.0 0.0.0.255 area 0
R1(config-router)# network 10.0.0.0 0.0.0.3 area 0
R1(config-router)# exit
R1(config)# do write memory
CLI Configuración OSPF en Router ISP (Borde)

Configuraremos OSPF en el router del proveedor. Al activarlo, verás cómo se genera la adyacencia de forma automática en la consola.

ISP> enable
ISP# configure terminal
# (Asegurate de borrar las rutas estáticas viejas si existen: no ip route 192.168...)

ISP(config)# router ospf 1
ISP(config-router)# router-id 2.2.2.2

# Anunciar su enlace WAN y la red pública del DNS (8.0.0.0/8 -> Wildcard 0.255.255.255)
ISP(config-router)# network 10.0.0.0 0.0.0.3 area 0
ISP(config-router)# network 8.0.0.0 0.255.255.255 area 0
ISP(config-router)# exit
ISP(config)# do write memory

Al cabo de unos instantes, la CLI del ISP disparará un mensaje de Syslog indicando éxito: %OSPF-5-ADJCHG: Process 1, Nbr 1.1.1.1 on GigabitEthernet0/0/0 from LOADING to FULL, Loading Done.

Fase 5: Comprobaciones de Adyacencia y Tabla OSPF

Validaremos que los routers hayan aprendido las redes remotas mediante software dinámico.

Verificación Comandos de Diagnóstico de Estado de Enlace

1. Verificar los Vecinos OSPF

Para confirmar que R1 e ISP establecieron una relación de confianza mutua, ejecutá:

R1# show ip ospf neighbor

Debés verificar que el estado figure como FULL/--- o FULL/DR, lo que certifica el intercambio completo de las bases de datos (LSDB).

2. Auditar la Tabla de Enrutamiento con Prefijo "O"

Ejecutá el comando para ver la tabla de enrutamiento del Router Corporativo R1:

R1# show ip route

Notarás que apareció una red remota aprendida dinámicamente con la letra **O** (OSPF):

O    8.0.0.0/8 [110/2] via 10.0.0.2, 00:04:12, GigabitEthernet0/0/3

Nota técnica: El valor 110 representa la Distancia Administrativa (DA) nativa de OSPF, utilizada por el router para medir la confiabilidad del protocolo.

3. Prueba del Plano de Datos

Realizá un ping o trazá la ruta desde cualquier PC local hacia la dirección 8.8.8.8. La conectividad debe ser exitosa, demostrando que el "tubo" lógico (Trunk) y el mapa dinámico (OSPF) operan en perfecta sintonía corporativa.
