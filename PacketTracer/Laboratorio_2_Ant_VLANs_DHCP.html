Laboratorio 2: Aislamiento de Capa 2 y DHCP Centralizado por Servidor Telecomunicaciones B

Este laboratorio práctico tiene como objetivo comprender el comportamiento de la segmentación de tráfico en la Capa 2 del modelo OSI sin la intervención de un router. Se desplegarán tres switches independientes para aislar los dominios de colisión y difusión. Dos redes utilizarán direccionamiento estático manual, mientras que la tercera contará con un Servidor genérico dedicado a la entrega dinámica de parámetros IP.

📌 Concepto Clave de Capa 2: Al no haber un router o switch de Capa 3 conectando las redes, las VLANs se encuentran completamente aisladas entre sí. Un equipo de la VLAN 10 no podrá comunicarse bajo ninguna circunstancia con uno de la VLAN 20 o 30, sirviendo este escenario como demostración pura de seguridad perimetral local.

Fase 1: Diseño de la Topología y Planificación IP

Estructuraremos el escenario usando exclusivamente hardware de conmutación de accesos y hosts finales.

Topología Componentes e Interconexiones

Componentes requeridos en el lienzo:

  • 3 Switches (S1, S2, S3): Modelo clásico 2960 de 24 puertos.
  • 3 PCs Finales (PC-V10, PC-V20, PC-V30): Distribuidas una en cada switch.
  • 1 Servidor Genérico (Server-DHCP): Acoplado al Switch 3.

Fase 2: Configuración de los Dispositivos

A continuación, se detallan los comandos de aprovisionamiento de las VLANs de acceso en los switches y los pasos lógicos para activar el servidor en Packet Tracer.

CLI Aislamiento de Puertos en Switches 2960

Switch S1 (Dominio Ingeniería)

Switch> enable
Switch# configure terminal
Switch(config)# hostname S1
S1(config)# vlan 10
S1(config-vlan)# name Ingenieria
S1(config-vlan)# exit
S1(config)# interface fastEthernet 0/2
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 10
S1(config-if)# do write memory

Switch S2 (Dominio Administración)

Switch> enable
Switch# configure terminal
Switch(config)# hostname S2
S2(config)# vlan 20
S2(config-vlan)# name Administracion
S2(config-vlan)# exit
S2(config)# interface fastEthernet 0/2
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 20
S2(config-if)# do write memory

Switch S3 (Dominio Invitados con Servidor)

Switch> enable
Switch# configure terminal
Switch(config)# hostname S3
S3(config)# vlan 30
S3(config-vlan)# name Invitados
S3(config-vlan)# exit
S3(config)# interface range fastEthernet 0/1 - 2
S3(config-if-range)# switchport mode access
S3(config-if-range)# switchport access vlan 30
S3(config-if-range)# do write memory
Servicios Configuración Visual del Server-DHCP

Para levantar el servidor sin usar comandos IOS, interactuaremos con la interfaz gráfica (GUI) del objeto Server en Packet Tracer:

Paso A: Direccionamiento Fijo de la Tarjeta del Servidor

Hacé clic en Server-DHCP, andá a Desktop > IP Configuration, seleccioná Static y cargá:

  • IP Address: 192.168.30.5
  • Subnet Mask: 255.255.255.0
  • Default Gateway / DNS: Dejar vacíos (0.0.0.0), ya que no contamos con salida de red local.

Paso B: Activación del Rol DHCP

  1. Dentro de las solapas del servidor, ingresá a Services (Servicios) y hacé clic en DHCP.
  2. Cambió el botón de Service a la posición de ON.
  3. Modificá los siguientes campos en el pool por defecto (serverPool):
    • Start IP Address: 192.168.30.10
    • Subnet Mask: 255.255.255.0
    • Maximum number of Users: 50
  4. Hacé clic de manera obligatoria en el botón Save (Guardar).

Fase 3: Pruebas de Comprobación y Comportamiento Esperado

Al ser un escenario puramente de Capa 2 segmentado, las pruebas deben validar que el DHCP responde de manera confinada y que no existe fuga de tráfico entre entornos.

Verificación Pruebas de Fuego en Computadoras Finales

Prueba 1: Verificación de Concesión en PC-V30

Hacé clic en PC-V30, andá a Desktop > IP Configuration y conmutá de Static a DHCP. Al cabo de unos segundos, la PC debe recibir exitosamente la dirección IP 192.168.30.10 proveniente de nuestro servidor.

Prueba 2: Comprobación de Aislamiento Total (Pings Fallidos Esperados)

Abrí la consola de comandos (Command Prompt) en la PC-V10 (IP 192.168.10.10) e intentá conectarte con los hosts de los otros switches:

C:\> ping 192.168.20.10    # Destinado a PC-V20. Resultado: Request Timed Out (Fallido)
C:\> ping 192.168.30.10    # Destinado a PC-V30. Resultado: Request Timed Out (Fallido)

Nota de examen: Este comportamiento de falla es **correcto y esperado**. Demuestra que las VLANs cumplen su función nativa de contener y bloquear las transmisiones por fuera de su ID de red.

Prueba 3: Comprobación Interna de Capa 2 (Ping Exitoso)

Desde la terminal de la PC-V30 (IP 192.168.30.10), tirá un ping hacia el servidor que le otorgó la dirección para confirmar la conectividad intracomunitaria:

C:\> ping 192.168.30.5     # Destinado al Server-DHCP. Resultado: Reply from 192.168.30.5 (Exitoso)
